# 玩家钓鱼小游戏设计

日期：2026-06-20

## 摘要

新增主世界玩家手动钓鱼玩法。玩家在主世界任意位置按 `F` 开始钓鱼，等待咬钩后在 1.5 秒内再次按 `F` 收杆，随后通过连续按 `F` 推进进度条。进度条满时钓鱼成功并获得 1 金币，进度降到 0 时失败。

本设计只实现玩家手动钓鱼，不新增鱼库存、鱼类图鉴、渔夫职业、渔屋建筑或自动捕鱼生产链。

## 目标

- 给主世界白天增加一个随时可做的低收益互动。
- 让前景水面成为玩法语义，不要求玩家站在桥边或水边。
- 保持系统轻量：成功直接获得金币，不引入新资源库存。
- 保持数据分离：概率、时间、进度、奖励等可调数值进入 `GameData.gd`。
- 避免继续扩大 `BuildManager.gd` 的职责，钓鱼逻辑使用独立管理脚本。

## 非目标

- 不在河湾商盟或其他游历场景接入钓鱼。
- 不新增鱼类物品、背包、售卖界面或价格波动。
- 不新增渔夫、渔屋、鱼市或自动捕鱼。
- 不改动现有桥梁、农田、母树、采石场等资源点交互规则。
- 不新增美术大资产；第一版 UI 使用 Godot 控件和简易色块。

## 玩家流程

1. 玩家在主世界正常状态下按 `F`。
2. 进入等待咬钩阶段，显示钓鱼状态提示和等待计时。
3. 每经过 1 秒进行一次咬钩判定：
   - 第 1 秒概率为 5%。
   - 第 2 秒概率为 10%。
   - 第 3 秒概率为 15%。
   - 第 4 秒概率为 20%。
   - 后续每秒继续增加 5%，最高值由数据层配置，第一版建议封顶 80%。
4. 咬钩后进入收杆提示阶段，显示 `咬钩了！1.5 秒内按 F 收杆`。
5. 玩家在 1.5 秒内按 `F`，进入拉杆进度阶段；未按则失败。
6. 拉杆进度初始为 20%。
7. 拉杆阶段每按一次 `F` 增加固定进度，同时进度持续下降。
8. 进度达到 100% 时成功，调用 `BuildManager.add_gold(1)` 增加 1 金币。
9. 进度降到 0 时失败。
10. 钓鱼任意阶段按 `Q` 取消。

## 输入规则

- `F`：开始钓鱼、咬钩后收杆、拉杆阶段推进进度。
- `Q`：取消钓鱼。
- 钓鱼期间不处理建造、拆除、建筑面板、修复、流浪汉交互等普通操作。
- 若暂停菜单打开、玩家死亡、场景不是主世界、测试面板正在操作，不能开始钓鱼。
- 若玩家在钓鱼过程中死亡、切换场景或打开暂停菜单，钓鱼立即取消。

## 状态机

钓鱼管理器维护以下状态：

- `idle`：未钓鱼。
- `waiting_for_bite`：等待咬钩，每秒进行概率判定。
- `bite_window`：已咬钩，等待玩家在 1.5 秒内按 `F`。
- `reeling`：拉杆进度阶段，连续按 `F` 增加进度，进度自动降低。
- `success` / `failed`：结算状态，显示短暂结果后回到 `idle`。

状态转换必须集中在钓鱼管理器中，其他系统只通过公开方法查询 `is_fishing()` 或请求取消。

## 数据层

在 `scripts/GameData.gd` 新增钓鱼配置，例如：

- `FISHING.bite_base_chance = 0.05`
- `FISHING.bite_chance_step = 0.05`
- `FISHING.bite_chance_max = 0.8`
- `FISHING.bite_check_seconds = 1.0`
- `FISHING.hook_window_seconds = 1.5`
- `FISHING.reel_start_progress = 0.2`
- `FISHING.reel_press_gain = 0.15`
- `FISHING.reel_decay_per_second = 0.18`
- `FISHING.reward_gold = 1`

脚本不得在逻辑中硬编码这些数值。若测试需要固定随机结果，钓鱼管理器应允许注入随机种子或使用可控规则函数。

## 组件设计

### FishingRules.gd

纯规则脚本，负责：

- 根据等待秒数返回当前咬钩概率。
- 根据 roll 判断是否咬钩。
- 推进拉杆进度并返回成功/失败条件。

### FishingManager.gd

主世界运行时节点，负责：

- 监听 `F` / `Q` 输入。
- 管理钓鱼状态机和计时。
- 创建和刷新钓鱼 UI。
- 成功时调用 `BuildManager.add_gold(GameData.fishing_value("reward_gold"))`。
- 钓鱼期间向 `BuildManager` 或输入流程暴露阻塞状态，避免同一按键触发其他交互。

### FishingUI

由 `FishingManager.gd` 运行时创建或作为 `Main.tscn` 子节点存在。第一版包括：

- 状态文本：等待、咬钩、拉杆、成功、失败。
- 咬钩倒计时文本。
- 拉杆进度条。

UI 显示在屏幕下方偏中位置，不遮挡已有底部建造栏核心文字。钓鱼结束后自动隐藏。

## 与现有系统的关系

- 金币奖励通过 `BuildManager.add_gold`，保证金币 UI 和建筑栏灰色状态自动刷新。
- 不进入存档。钓鱼是短时动作，自动存档不保存钓鱼中状态；读档后始终处于 `idle`。
- 不改变死亡复活、夜袭、怪物、NPC、桥梁、农田和资源点规则。
- 若后续加入鱼库存或渔夫职业，本系统可保留为玩家手动捕鱼入口。

## 错误处理

- 找不到 `BuildManager` 时，钓鱼成功只结束状态并打印警告，不崩溃。
- 找不到 UI 容器时，运行时创建必要 UI。
- 配置值异常时使用安全范围，例如概率限制在 `0.0` 到 `1.0`，进度限制在 `0.0` 到 `1.0`。

## 测试计划

- `FishingRulesTest.gd`
  - 第 1/2/3/4 秒概率分别为 5%/10%/15%/20%。
  - 概率按每秒 5% 增加，并不会超过上限。
  - roll 低于概率时咬钩，等于或高于概率时不咬钩。
  - 拉杆进度按按键增加、按时间降低，达到 100% 成功，降到 0 失败。

- `FishingManagerTest.gd`
  - 主世界按 `F` 从 idle 进入等待咬钩。
  - 咬钩后 1.5 秒内按 `F` 进入拉杆阶段。
  - 咬钩后超过 1.5 秒未按 `F` 失败。
  - 拉杆阶段连续按 `F` 可成功并增加 1 金币。
  - 拉杆阶段不按键会失败。
  - 任意阶段按 `Q` 取消并回到 idle。
  - 玩家死亡或暂停时钓鱼取消。

- `GameDataSeparationTest.gd`
  - 覆盖钓鱼概率、时间、进度和奖励金币来自 `GameData.gd`。

- `GAME_REFERENCE.txt`
  - 记录新增按键 `F`、新增钓鱼 UI、`FishingRules.gd`、`FishingManager.gd`、钓鱼奖励和限制。

## 审查结论

- 无占位项。
- 范围聚焦在玩家手动钓鱼，不包含库存和职业生产链。
- 数值和规则均有数据层入口。
- 与现有 `E` 建造/交互规则解耦，使用 `F` 作为钓鱼主键。
